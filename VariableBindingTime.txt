1. Создание событий - инцидентов.
Есть 2 типа инцидентов. В разных участках программы при создании инцидента тип инициализируется одним из двух значений строкового типа;
Значения вынесены в константы для связывания на этапе компиляции

static final String DEFAULT_INCIDENT_TYPE = "default";
static final String CUSTOM_INCIDENT_TYPE = "custom";

С таким решением в случае изменения наименования типа нужно будет внести изменение только в 1 месте.

2. Вынесение параметров в конфигурационный файл, который считывается при начальной загрузке программы.
Эти параметры изменяются редко, их изменение должно привести к перезапуску приложения.
yaml - конфиг:

default-resolver:
   sub: "00156"
   channel: "chanel"
local-mode: true
